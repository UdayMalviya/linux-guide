Security of Linux

Linux is considered one of the most secure operating systems due to its open-source model, permission system, and active community maintenance.

Key Security Features

  • User/Group Permissions: Every file has Owner, Group, Others permissions (rwx). Prevents unauthorized access.
  • Root vs Normal User: Root has full control; normal users have restricted access, limiting damage from attacks.
  • PAM (Pluggable Authentication Modules): Flexible authentication framework for applications.
  • SELinux / AppArmor: Mandatory Access Control (MAC) policies that restrict processes beyond normal permissions.
  • Firewall (iptables / nftables): Packet filtering at kernel level. Controls inbound/outbound traffic.
  • SSH (Secure Shell): Encrypted remote login. Replaces insecure Telnet/FTP.
  • File Encryption: LUKS for disk encryption, GPG for files.
  • Audit Logging: /var/log/ captures all system events. Tools: auditd, journalctl.
  • Open Source Advantage: Code is reviewed by millions — vulnerabilities are found and patched faster.
  • Minimal Attack Surface: Only required services run; fewer background processes = fewer vulnerabilities.
Command to check failed logins: cat /var/log/auth.log | grep "Failed"

Linux Architecture

Layer Descriptions

  • Hardware: Physical components — CPU, RAM, storage, network cards.
  • Kernel: Core of Linux. Manages hardware, processes, memory, file systems, and device drivers directly.
  • System Call Interface: Bridge between user programs and kernel. Programs request kernel services via system calls.
  • Shell: Command-line interpreter. Takes user commands, translates them, and passes them to the kernel via system calls.
  • User Applications: Programs users run — text editors, browsers, compilers, scripts.

Rules of Writing File Names in Linux

  • File names are case-sensitive: File.txt ≠ file.txt
  • Maximum length: 255 characters per file name; 4096 for full path.
  • Can contain letters, digits, underscore _, hyphen -, dot .
  • Avoid special characters: / \ * ? < > | ; & $ ! # @ [ ] ( ) { }
  • Avoid spaces in file names (use _ or - instead); if needed, use quotes: "my file.txt"
  • File names starting with . (dot) are hidden files: .bashrc, .ssh
  • No reserved names like / (root) or . and .. (current & parent dir)
  • Extensions are optional — Linux does not rely on file extensions to determine file type
  • Avoid starting with - (hyphen) — can be misinterpreted as command options
Good names: report_2024.txt, my-script.sh  |  Bad names: my file!.txt, -report

Rules for Having a Password in Linux

  • Minimum 8 characters (PAM default; configurable)
  • Should include: uppercase, lowercase, digits, special characters
  • Must NOT be a dictionary word or the username
  • Must NOT be a previously used password (password history via PAM)
  • Password expiry can be set using chage -M 90 username (expires in 90 days)
  • Stored as hashed values in /etc/shadow (never in plain text)
  • Root can change any user's password: passwd username
  • Users change own password: passwd (no argument)
  • Account lockout after N failed attempts (configured in PAM: pam_faillock)

Password Fields in /etc/shadow

username:$6$hash...:18900:0:99999:7:::

Fields: username : hashed-password : last-change : min-age : max-age : warn-days : inactive : expire

Hash types: $1=MD5, $5=SHA-256, $6=SHA-512 (most secure)

Local and Global Environment Variables and Their Difference

Environment Variable: A named value stored in memory that affects the behaviour of processes and programs.

  • Exist only in the current shell session
  • Not inherited by child processes

Multiprogramming, Multitasking, and Multiuser in Linux

TermDefinitionLinux Example
Multiprogramming Multiple programs are loaded in memory at the same time. CPU switches between them when one is waiting for I/O. Goal: maximize CPU utilization. Linux keeps many processes in memory (browser, text editor, daemon) — CPU picks whichever is ready to run.
Multitasking CPU rapidly switches between tasks giving illusion of simultaneous execution. Each task gets a time slice (time-sharing / preemptive scheduling). You can compile code, play music, and browse simultaneously on Linux. Kernel scheduler allocates CPU slices.
Multiuser Multiple users can log in and use the system simultaneously. Each user has separate processes, files, and permissions. A Linux server can have 100s of users logged in via SSH, each running their own programs independently.
Linux uses the Completely Fair Scheduler (CFS) for multitasking. Commands: top, htop, who, w

What is AWK? List Any 2 Features

AWK is a powerful text-processing language/utility in Linux used for pattern scanning, data extraction, and reporting. Created by Aho, Weinberger, and Kernighan.

Syntax

awk 'pattern { action }' filename

Example

awk '{print $1, $3}' file.txt       # Print 1st and 3rd column
awk -F: '{print $1}' /etc/passwd    # Print usernames (: separator)
awk '/error/ {print}' log.txt       # Print lines with "error"
awk '{sum += $2} END {print sum}' data.txt   # Sum column 2

2 Key Features

  1. Field and Record Processing: AWK automatically splits each line into fields ($1, $2…) using a delimiter. This makes processing structured data (like CSV, logs, /etc/passwd) very easy without manual parsing.
  2. Built-in Variables: AWK provides powerful built-in variables: NR (current record/line number), NF (number of fields), FS (field separator), OFS (output field separator), BEGIN/END blocks for initialization and finalization.

Responsibilities of the Shell

  1. Command Interpretation: Reads and parses commands typed by the user; determines if it's a built-in command or an external program.
  2. Program Execution: Forks a child process and uses exec() system call to run the requested program.
  3. I/O Redirection: Handles > (output), < (input), >> (append) to redirect streams.
  4. Pipe Management: Connects output of one command to input of another using |. e.g., ls | grep .txt
  5. Variable Management: Stores and retrieves shell variables and environment variables.
  6. Wildcard Expansion: Expands wildcards like *, ?, [] before passing to commands (globbing).
  7. Shell Scripting: Executes scripts containing loops, conditions, functions — automation of tasks.
  8. Job Control: Manages foreground/background processes. Ctrl+Z suspends, bg/fg switches, & backgrounds.
  9. Signal Handling: Intercepts and handles signals like SIGINT (Ctrl+C), SIGTERM.
  10. Command History: Maintains command history (~/.bash_history) for recall and reuse.

Protection Modes of Linux OS

1. Dual Mode Operation

The CPU operates in two modes. User mode (limited privileges) for user applications; Kernel mode (full privilege) for OS operations. This prevents user programs from directly accessing hardware or critical memory.

2. I/O Protection

All I/O instructions are privileged instructions — only the OS (kernel mode) can execute them. If a user program attempts direct I/O, it causes a trap (exception), and the OS terminates or handles it. User programs must request I/O via system calls.

3. Memory Protection

Linux uses base and limit registers plus virtual memory (paging/segmentation) to ensure each process can only access its own memory space. Any attempt to access another process's memory causes a segmentation fault. Hardware enforces this via the MMU (Memory Management Unit).

4. Hardware Protection

Direct hardware access is restricted to the kernel. Peripherals (disk, NIC, GPU) are managed by device drivers running in kernel space. User programs communicate with hardware only through well-defined system calls, preventing hardware damage or unauthorized access.

5. CPU Protection

A timer (hardware clock) is set by the OS before giving CPU to a user process. When the timer expires, a hardware interrupt fires, the OS regains control, and switches to another process. This prevents any single process from monopolizing the CPU (infinite loops can't hang the system).

NFS – Network File System

NFS (Network File System) is a distributed file system protocol developed by Sun Microsystems that allows a computer to access files over a network as if they were on its local disk.

How It Works

An NFS server exports a directory. NFS clients mount that directory locally. The client can then read/write files transparently over the network using RPC (Remote Procedure Calls).

# On Server – export /shared
echo "/shared  192.168.1.0/24(rw,sync)" >> /etc/exports
exportfs -ra

# On Client – mount server's share
mount server_ip:/shared /mnt/nfs

GRUB Configuration File

GRUB (Grand Unified Bootloader) is the default bootloader in Linux. It presents the boot menu and loads the selected OS kernel into memory.

Key Config File: /boot/grub/grub.cfg

This file is auto-generated — do not edit directly. Edit /etc/default/grub then run update-grub.

/etc/default/grub – Important Parameters

# /etc/default/grub
GRUB_DEFAULT=0              # Default menu entry (0 = first)
GRUB_TIMEOUT=5              # Seconds to show menu before boot
GRUB_DISTRIBUTOR="Ubuntu"   # OS name in boot menu
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"  # Kernel boot params
GRUB_CMDLINE_LINUX=""       # Extra kernel parameters
GRUB_DISABLE_RECOVERY=true  # Hide recovery mode entries
GRUB_GFXMODE=1024x768       # Screen resolution for menu

Apply Changes

sudo update-grub        # Regenerate /boot/grub/grub.cfg
sudo grub-install /dev/sda  # Reinstall GRUB to disk MBR

GRUB Boot Process

BIOS/UEFI → GRUB Stage 1 (MBR) → GRUB Stage 2 → /boot/grub/grub.cfg → Kernel loaded → initramfs → init/systemd

CIDR – Classless Inter-Domain Routing

CIDR is a method for allocating IP addresses and routing that replaces the old Class A/B/C system. It uses a prefix length (slash notation) to define the network portion of an IP address.

Notation

IP Address / Prefix-Length
Example: 192.168.1.0/24

The /24 means the first 24 bits are the network part; remaining 8 bits are for hosts.

Examples

10.0.0.0/8255.0.0.016,777,21410.0.0.1 – 10.255.255.254
172.16.0.0/16255.255.0.065,534172.16.0.1 – 172.16.255.254
192.168.1.0/28255.255.255.24014192.168.1.1 – .14

Hosts = 2^(32-prefix) − 2 (subtract network address and broadcast address)
CIDR allows variable-length subnetting, reducing IP address waste compared to classful networking.

Process to Set Up a Network Interface Card (NIC)

Setting up a NIC involves configuring an IP address, gateway, DNS, and activating the interface so the system can communicate on the network.

Method 1: Temporary (using ip / ifconfig)

# Assign IP address
ip addr add 192.168.1.100/24 dev eth0

# Bring interface up
ip link set eth0 up

# Add default gateway
ip route add default via 192.168.1.1

# Verify
ip addr show eth0

Method 2: Permanent (using /etc/network/interfaces – Debian/Ubuntu)

# Edit /etc/network/interfaces
auto eth0
iface eth0 inet static
    address 192.168.1.100
    netmask 255.255.255.0
    gateway 192.168.1.1
    dns-nameservers 8.8.8.8 8.8.4.4

# Restart networking
systemctl restart networking

Method 3: NetworkManager (nmcli – modern Linux)

nmcli con add type ethernet ifname eth0
nmcli con mod eth0 ipv4.addresses 192.168.1.100/24
nmcli con mod eth0 ipv4.gateway 192.168.1.1
nmcli con mod eth0 ipv4.dns "8.8.8.8"
nmcli con mod eth0 ipv4.method manual
nmcli con up eth0

Verify Configuration

ip addr         # Show interfaces and IPs
ip route        # Show routing table
ping 8.8.8.8    # Test connectivity

Brief Notes on: Dual Kernel · Booting · FTP · Java Client · Multiprocessing · RPM

Dual Kernel

A system where two kernels coexist — typically a real-time kernel alongside the standard Linux kernel (e.g., PREEMPT-RT patch or Xenomai). Used in embedded systems and industrial applications where real-time guarantees are needed alongside general-purpose tasks.

Booting

The process of starting the computer and loading the OS. Linux boot sequence:

Power ON → BIOS/UEFI (POST) → MBR/GPT → Bootloader (GRUB) → Kernel loaded → initramfs → init / systemd → Login prompt

Key files: /boot/vmlinuz (kernel), /boot/initrd.img (initial RAM disk), /etc/inittab or systemd units.

FTP (File Transfer Protocol)

Network protocol for transferring files between client and server on port 21 (control) and port 20 (data). Commands: get (download), put (upload), ls (list). FTP is insecure (plain text); use SFTP or FTPS for encrypted transfers.

Java Client

A Java application that connects to a server (using sockets, HTTP, RMI, or JMS). On Linux, Java clients run on the JVM. Example: a Java app using Socket class to connect to a server, send/receive data. To run: java -jar client.jar. Java's WORA (Write Once Run Anywhere) makes it ideal for cross-platform Linux clients.

Multiprocessing

The ability to run processes on multiple CPU cores/processors simultaneously (true parallelism). Linux supports SMP (Symmetric Multi-Processing) — all CPUs share a single memory space and are managed by one OS instance. Different from multitasking (single CPU, time-sliced). Linux distributes processes across cores automatically. Tools: nproc, lscpu, htop.

RPM (Red Hat Package Manager)

Package management system used in Red Hat-based distros (RHEL, CentOS, Fedora). An RPM file contains a compiled application + metadata (name, version, dependencies).

rpm -ivh package.rpm    # Install
rpm -Uvh package.rpm    # Upgrade
rpm -evh package        # Remove (erase)
rpm -qa                 # List all installed packages
rpm -qi package         # Query info about package

UID and SGID

UID – User ID

A numeric identifier assigned to each user in Linux. Used by the kernel to track ownership and permissions.
  • Root UID = 0 (superuser, full system access)
  • System users: UID 1–999 (daemons, services)
  • Regular users: UID 1000+ (created by admin)
  • View UIDs: cat /etc/passwd or id username
  • The kernel uses UID (not username) for all permission checks

SGID – Set Group ID

A special permission bit. When set on a file/directory, it causes processes or new files to inherit the group owner, not the user's primary group.

SGID on Files

When SGID is set on an executable, the process runs with the file's group permissions, not the user's group. Useful for programs that need group-level access.

SGID on Directories

Files created inside the directory inherit the directory's group, not the creator's primary group. Ideal for shared team directories.

# Set SGID on directory
chmod g+s /shared_dir
# or
chmod 2755 /shared_dir

# View SGID: 's' appears in group execute position
ls -l /shared_dir
# Output: drwxr-sr-x  (s = SGID set)
SUID (Set User ID) is similar but applies to user instead of group. Example: /usr/bin/passwd has SUID so regular users can change their password (which requires writing to /etc/shadow).

What is the Shell in Linux?

Shell is a special user program (command interpreter) that provides an interface between the user and the Linux kernel. It accepts commands from the user (or scripts), interprets them, and causes the kernel to execute the requested operations.

Role of Shell

  • Sits between the user and kernel, acting as a mediator
  • Reads input from keyboard/script; sends instructions to kernel via system calls
  • Displays output back to the user (terminal/screen)
  • Provides a scripting language for automation

Types of Shell

  • Interactive Shell: Takes commands directly from user (terminal). e.g., login shell after SSH.
  • Non-interactive Shell: Runs scripts without user input. e.g., cron jobs, startup scripts.
  • Login Shell: Started at login; reads /etc/profile, ~/.bash_profile.
  • Non-login Shell: Opened in existing session; reads ~/.bashrc.

How Each Shell Processes a Command

  1. Tokenization: Shell splits command line into tokens (command, arguments, operators).
  2. Expansion: Variables ($VAR), wildcards (*), command substitution ($(cmd)) are expanded.
  3. Parsing: Shell determines if it's a built-in command (handled internally) or external program.
  4. Execution: For external commands, shell calls fork() then exec() to run the program.
  5. I/O Redirection: Shell sets up pipes/redirections before handing control to the process.
  6. Wait: Shell waits (for foreground) or continues (for background &) for the process to complete.
For exam: Bash is the most important shell to know. It is the default in most Linux distributions and the most commonly used for scripting.
